This page is located at /index.html and should be the opening page of the web app.

Links to files in subfolders should work without a relative path prefix (i.e. current url on target page should contain index.html): subfolder/index.html

Links to files in subfolders should work with a relative path prefix (i.e. current url on target page should contain index.html): ./subfolder/index.html

Links to subfolders without a relative path prefix and with a trailing slash should serve the contexts of index.html (i.e. current url on target page should not contain index.html): subfolder/

Links to subfolders with a relative path prefix and with a trailing slash should serve the contexts of index.html (i.e. current url on target page should not contain index.html): ./subfolder/

Links to subfolders without a relative path prefix and without a trailing slash should redirect to having a trailing slash which should then behave as above (i.e. current url on target page should not contain index.html): subfolder

Links to subfolders with a relative path prefix and without a trailing slash should redirect to having a trailing slash which should then behave as above (i.e. current url on target page should not contain index.html): ./subfolder

Note: Redirect of a subdirectory from no trailing slash (/subfolder) to a trailing slash (/subfolder/) is important for relative paths to resolve correctly on that target page. Consider ./image.jpg which without a trailing slash resolves to /image.jpg while with a trailing slash resolves to /subfolder/image.jpg.
